# Design Leyu Hub

这是一个用于归档和发布多个独立 HTML 页面的仓库。仓库内包含各类静态页面资源，不针对特定域名或网站，主要作为设计稿、原型页面、实验性界面或功能演示的集中存放地。

## 项目简介

本仓库旨在提供一个轻量、开放的 HTML 页面集合。每个页面均为独立文件，可直接在浏览器中打开查看。仓库适合存放：

- 设计稿预览页面
- 交互原型演示
- 前端功能测试页
- 静态信息展示页

不包含后端逻辑或数据库依赖，所有内容均为纯前端静态资源。

## 目录说明

```
design-leyu-hub/
├── pages/          # 存放所有独立 HTML 页面
├── assets/         # 公共资源（图片、样式、脚本等）
├── docs/           # 文档或说明文件（可选）
└── README.md       # 本文件
```

- `pages/` 目录下每个 HTML 文件代表一个独立页面，文件名应简洁且具描述性。
- `assets/` 用于存放页面共享的静态资源，如通用样式表、JavaScript 库或图片。
- `docs/` 可存放与页面相关的说明文档或设计参考。

## 页面归档说明

- 每个页面均为自包含的 HTML 文件，尽量将样式与脚本内联或引用自 `assets/`。
- 页面命名建议使用英文小写字母、数字和连字符，例如 `demo-landing-page.html`。
- 如需引用外部资源（如字体、第三方库），请注明来源，并确保符合使用许可。
- 页面内容应保持整洁，避免包含未注释的测试代码或敏感信息。

## 维护说明

- 欢迎提交 Pull Request 添加新的页面或改进现有页面。
- 新增页面时，请在 `pages/` 下创建新文件，并确保在本地测试无误。
- 如涉及资源更新，请同步更新 `assets/` 中的对应文件。
- 不建议在仓库中存放过大的文件（如高清视频或大型压缩包），以免影响仓库克隆速度。
- 本仓库不主动收录涉及商业推广、恶意内容或违反法律法规的页面。

## 使用方式

1. 克隆仓库到本地：
   ```bash
   git clone https://github.com/your-username/design-leyu-hub.git
   ```
2. 直接在浏览器中打开 `pages/` 下的任意 HTML 文件即可查看。
3. 也可通过 GitHub Pages 或其他静态托管服务部署后在线访问。

## 贡献指南

如果你希望贡献页面，请遵循以下步骤：

1. Fork 本仓库。
2. 在 `pages/` 下添加你的 HTML 文件。
3. 如有必要，在 `assets/` 中添加相关资源。
4. 提交 Pull Request，并简要描述页面用途。
